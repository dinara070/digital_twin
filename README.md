# Digital Twin v2.0

Розширена версія проєкту цифрового двійника. Порівняно з початковою версією
додано три великі блоки можливостей:

## 🆕 Що нового

### 1. Реальний LLM (Anthropic API)
- `digital_twin/llm.py` — `LLMProvider` викликає справжній Claude через Anthropic
  Messages API замість шаблонних фраз.
- `CognitiveEngine` будує системний промпт із конфігурації особистості + релевантних
  спогадів (RAG) + історії розмови, і **автоматично відкочується на шаблонний режим**,
  якщо API-ключ не заданий або стався збій — додаток ніколи не падає через LLM.
- Ключ вводиться в UI (бокова панель → «Налаштувати LLM») і зберігається
  **зашифрованим** у SQLite через `EncryptionManager`.

### 2. Реальні embeddings замість хеш-заглушки
- `digital_twin/memory.py` тепер підтримує три бекенди пошуку по пам'яті з
  автоматичним fallback:
  1. `sentence-transformers` (справжні семантичні embeddings, якщо встановлено)
  2. `scikit-learn` TF-IDF (статистично обґрунтовані embeddings, без потреби
     завантажувати ваги моделі — використовується за замовчуванням)
  3. Хеш-based fallback (як у первинній версії — крайній випадок)

### 3. Персистентність та багато профілів
- `digital_twin/db.py` — SQLite-шар (`TwinDatabase`), що зберігає особистість,
  спогади (разом із векторами), історію розмов і емоцій, налаштування спадщини
  та зашифрований API-ключ — усе прив'язано до `profile_id`.
- У Streamlit-додатку можна створювати кілька профілів двійника, перемикатися
  між ними, і стан відновлюється після перезапуску застосунку.

### 4. Аналітика
- `digital_twin/analytics.py` + нова вкладка «📊 Аналітика» в UI: розподіл емоцій,
  джерела спогадів, активність за днями, найчастотніші слова, співвідношення
  LLM/шаблонних відповідей.

### 5. Дрібні покращення
- Виправлено баг, через який `EmotionalState` не записував нейтральні репліки
  в історію емоцій.
- Видалення окремих спогадів з UI.
- Повний експорт/імпорт стану двійника (JSON) — можна перенести профіль між
  інсталяціями.
- `LegacyProtocol` тепер підтримує реальний тригер за днями неактивності.

## 📁 Структура проєкту

```
digital_twin_project/
├── app.py                        # Streamlit-додаток
├── requirements.txt
└── digital_twin/
    ├── __init__.py                # публічний API пакета
    ├── security.py                # шифрування, доступ, протокол спадщини
    ├── memory.py                  # embeddings + векторна пам'ять (RAG)
    ├── personality_model.py       # конфіг особистості + емоційний стан
    ├── llm.py                     # Anthropic API + когнітивний двигун
    ├── identity.py                # голос / 3D-аватар / мова тіла (інтерфейси-заглушки)
    ├── db.py                      # SQLite-персистентність (багато профілів)
    ├── orchestrator.py            # координація всіх модулів
    ├── interfaces.py              # Web / Telegram / VR інтерфейси
    └── analytics.py               # обчислення статистики для дашборду
```

## 🚀 Запуск

```bash
pip install -r requirements.txt
# опційно, для семантичних embeddings замість TF-IDF:
# pip install sentence-transformers

streamlit run app.py
```

API-ключ Anthropic можна:
- ввести прямо в UI (бокова панель → «Налаштувати LLM»), або
- задати змінною середовища `ANTHROPIC_API_KEY` перед запуском.

## ⚠️ Про демо-реалізації, які варто замінити перед продакшн-використанням

- `EncryptionManager` використовує XOR-потоковий шифр для демонстрації.
  Для реального застосунку замініть на `cryptography.fernet.Fernet` або AES-GCM.
- `VoiceCloning` та `Avatar3D` — інтерфейси-заглушки, що показують, куди
  підключити реальні API (напр. ElevenLabs, Unreal MetaHuman).
- Аутентифікація паролем зберігає лише хеш у пам'яті процесу — для продакшну
  потрібна повноцінна система користувачів.
